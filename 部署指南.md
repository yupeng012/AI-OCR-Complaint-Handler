## 🎯 方案一：Streamlit Cloud（推荐新手）

### 优势
- ✅ 完全免费
- ✅ 一键部署
- ✅ 自动 HTTPS
- ✅ 无需维护服务器

### 步骤

#### 1. 准备 GitHub 仓库
```bash
cd /Users/wtueeq/AI-OCR-Complaint-Handler

# 初始化 Git（如果还没初始化）
git init
git add .
git commit -m "Initial commit - AI Complaint Handler"

# 推送到 GitHub
git remote add origin https://github.com/YOUR_USERNAME/AI-OCR-Complaint-Handler.git
git branch -M main
git push -u origin main
```

#### 2. 连接 Streamlit Cloud
1. 访问 https://streamlit.io/cloud
2. 点击 "Sign up with GitHub"
3. 选择你的仓库：`AI-OCR-Complaint-Handler`
4. 点击 "Deploy!"

#### 3. 配置自定义域名
1. 在 Streamlit Cloud 控制台 → Settings → Domain
2. 输入：`yupeng-ai.cn`
3. 按照提示配置 DNS：
   - 添加 CNAME 记录：`yupeng-ai.cn` → `YOUR-APP.streamlit.app`

#### 4. 配置环境变量
在 Streamlit Cloud 控制台 → Secrets 中添加：
```toml
[telegram]
bot_token = "YOUR_BOT_TOKEN"
chat_id = "YOUR_CHAT_ID"
```

## 🖥️ 方案二：VPS 部署（生产环境）

### 前提条件
- 一台 VPS 服务器（Ubuntu 20.04+）
- 域名 `yupeng-ai.cn` 已解析到服务器 IP
- SSH 访问权限

### 一键部署脚本

**在服务器上执行：**

```bash
# 1. 克隆部署脚本
cd /tmp
curl -O https://raw.githubusercontent.com/YOUR_USERNAME/AI-OCR-Complaint-Handler/main/deploy-to-yupeng.sh
chmod +x deploy-to-yupeng.sh

# 2. 编辑脚本中的变量
nano deploy-to-yupeng.sh
# 修改：EMAIL="your-email@example.com"
# 修改：GITHUB_REPO="你的仓库地址"

# 3. 执行部署
./deploy-to-yupeng.sh
```

### 手动部署步骤

如果一键脚本不可用，手动执行：

```bash
# 1. 安装系统依赖
sudo apt update
sudo apt install -y python3-pip python3-venv nginx certbot python3-certbot-nginx

# 2. 克隆应用
cd /var/www
sudo git clone https://github.com/YOUR_USERNAME/AI-OCR-Complaint-Handler.git
sudo chown -R www-data:www-data AI-OCR-Complaint-Handler

# 3. 设置 Python 环境
cd /var/www/AI-OCR-Complaint-Handler
sudo -u www-data python3 -m venv venv
sudo -u www-data ./venv/bin/pip install --upgrade pip
sudo -u www-data ./venv/bin/pip install -r requirements.txt

# 4. 创建 systemd 服务
sudo tee /etc/systemd/system/ai-complaint.service > /dev/null << 'EOF'
[Unit]
Description=AI Complaint Handler
After=network.target

[Service]
Type=exec
User=www-data
WorkingDirectory=/var/www/AI-OCR-Complaint-Handler
ExecStart=/var/www/AI-OCR-Complaint-Handler/venv/bin/python -m streamlit run app/unified_app.py --server.headless true --server.port 8501 --server.address 127.0.0.1
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable ai-complaint
sudo systemctl start ai-complaint

# 5. 配置 Nginx
sudo tee /etc/nginx/sites-available/yupeng-ai.cn > /dev/null << 'EOF'
server {
    listen 80;
    server_name yupeng-ai.cn www.yupeng-ai.cn;

    location / {
        proxy_pass http://127.0.0.1:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/yupeng-ai.cn /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx

# 6. 配置 SSL（Let's Encrypt）
sudo certbot --nginx -d yupeng-ai.cn -d www.yupeng-ai.cn --non-interactive --agree-tos --email your-email@example.com

# 7. 验证部署
sudo systemctl status ai-complaint
sudo systemctl status nginx
curl https://yupeng-ai.cn
```

## 🔒 安全建议

1. **防火墙配置**
```bash
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable
```

2. **定期更新**
```bash
sudo apt update && sudo apt upgrade -y
```

3. **监控异常登录**
```bash
sudo last
sudo grep "Failed password" /var/log/auth.log
```

4. **SSL 证书自动续期**
```bash
# Certbot 会自动续期，检查状态：
sudo certbot certificates
sudo systemctl status certbot.timer
```

## 📈 性能优化

### 1. 启用 Gzip 压缩
```nginx
# 在 nginx.conf 的 http 块中添加
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css text/xml text/javascript 
         application/x-javascript application/xml+rss 
         application/json application/javascript;
```

### 2. 配置缓存
```nginx
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

### 3. 增加工作进程
```nginx
# /etc/nginx/nginx.conf
worker_processes auto;
worker_connections 1024;
```
